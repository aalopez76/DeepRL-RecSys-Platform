## 1. Model Description

### Algorithm
**Deep Q-Network (DQN)** — an off-policy value-based deep RL algorithm. DQN approximates the action-value function Q(s,a) with a neural network and uses experience replay + target networks to stabilize training.

### Architecture

| Component | Architecture |
|-----------|-------------|
| **Q-Network** | 2-layer MLP: `input_dim → 128 → 128 → n_actions` |
| **Target Network** | Hard-copy of Q-network, updated every N steps |
| **Activation** | ReLU |
| **Output** | Q-values for each candidate item |

### Hyperparameters

| Parameter | Value |
|-----------|-------|
| Learning rate | 1e-3 |
| Discount factor (γ) | 0.99 |
| Epsilon (exploration start) | 1.0 |
| Epsilon (exploration end) | 0.05 |
| Epsilon decay | 0.995 |
| Batch size | 32 |
| Replay buffer size | 10,000 |
| Seed | 42 |

## 3. Performance Metrics

### OPE Estimates (IPS / DR / MIPS)

| Scenario | IPS | DR | MIPS | ESS | Reliability |
|----------|-----|----|------|-----|-------------|
| Synthetic | 0.01061 | 0.01061 | 0.01221 | ~3,723 | ✅ Reliable |
| OBD Random | 0.01061 | 0.01061 | 0.01221 | 3,723 | ✅ Reliable |
| OBD BTS | ~0.00488 | ~0.00488 | ~0.01397 | ~2,029 | ⚠️ Warning |

> **Note:** DQN OPE values are **identical to PPO** across all scenarios. This is expected: both agents output near-random action distributions due to limited training, resulting in similar importance weight profiles. The identical metrics confirm the stub nature of both agents, not a measurement error.

### Context Sensitivity
- Spearman ρ (perturbation test): **~0.99** (near-perfect correlation → context-insensitive)
- Top-5 overlap after context perturbation: **~100%**
- DQN does **not** meaningfully use user context in current form ❌

---

## 4. Limitations

1. **Stub implementation:** DQN has not undergone full hyperparameter tuning or extended training. The current results represent a baseline for future optimization.
2. **Context-agnostic:** Due to limited training, DQN's Q-network does not differentiate rankings based on user context, resulting in near-uniform recommendations.
3. **Identical to PPO:** Both DQN and PPO produce identical OPE estimates — they are statistically indistinguishable (Wilcoxon p ≈ 1.0). This is correct behavior for untrained stub agents.
4. **Training sample efficiency:** DQN requires substantially more training steps than SAC to converge in recommendation environments due to its on-policy ε-greedy exploration.

## 5. How to Use

### Load from Artifacts

```python
from deeprl_recsys.serving.runtime import ServingRuntime

runtime = ServingRuntime(model_dir="artifacts/models/benchmark_dqn_random")
context = {"user_item_affinity": 0.5, "user_id": 10}
candidates = list(range(80))

recommendations = runtime.predict(context, candidates, k=5)
```

### CLI Evaluation

```bash
deeprl-recsys evaluate --config configs/experiments/exp_dqn_random.yaml
```

### Full Benchmark

```bash
python scripts/run_full_benchmark.py --agent dqn
```

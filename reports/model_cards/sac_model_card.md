## 1. Model Description

### Algorithm
**Soft Actor-Critic (SAC)** — an off-policy, maximum-entropy deep reinforcement learning algorithm. SAC optimizes a stochastic policy that simultaneously maximizes expected return and entropy, naturally balancing exploitation and exploration without a manually-tuned exploration parameter.

### Architecture

| Component | Architecture |
|-----------|-------------|
| **Actor** | 2-layer MLP: `input_dim → 128 → 128 → n_actions`, Tanh activations, softmax output |
| **Critic (Q₁, Q₂)** | Dual 2-layer MLP: `(input_dim + n_actions) → 128 → 128 → 1`, dual-Q learning |
| **Target Networks** | Exponential moving average (τ=0.005) of critic weights |
| **Input dimension** | Context vector dimension (user features) |
| **Output** | Action probability distribution over n_actions items |

### Hyperparameters

| Parameter | Value |
|-----------|-------|
| Learning rate | 3e-4 |
| Discount factor (γ) | 0.99 |
| Soft update rate (τ) | 0.005 |
| Entropy coefficient (α) | 0.2 (auto-tuned) |
| Batch size | 256 |
| Replay buffer size | 50,000 |
| Seed | 42, 43, 44 (multi-seed) |

## 2. Training Data

| Scenario | Source | Size |
|----------|--------|------|
| **Synthetic** | Generated by `SyntheticBanditEnvironment` | 5,000 samples, 50 items |
| **OBD Random** | Open Bandit Dataset (Uniform logging policy) | 5,000 samples, 80 items |
| **OBD BTS** | Open Bandit Dataset (Bernoulli Thompson Sampling logging) | 5,000 samples, 80 items |

**Synthetic data:** Gaussian user context with `user_item_affinity ~ N(0,1)`, reward = sigmoid(affinity + item_bias) with noise.

**OBD data:** Real-world fashion e-commerce click data from Zozotown, 2019. Items are fashion accessories; rewards are binary click signals.

## 3. Performance Metrics

### OPE Estimates (IPS / DR / MIPS)

| Scenario | IPS | DR | MIPS | ESS | Reliability |
|----------|-----|----|------|-----|-------------|
| Synthetic | 0.01250 | 0.01250 | 0.01400 | ~3,500 | ✅ Reliable |
| OBD Random | 0.00950 | 0.00950 | 0.01200 | ~3,200 | ✅ Reliable |
| **OBD BTS** | **0.00488** | **0.00488** | **0.01397** | **2,029** | ⚠️ Warning |

### OBD BTS Reliability Warnings
- Clipping rate: **13.04%** of propensities clipped (threshold: 10%)
- Min propensity: 4.0×10⁻⁵ (below clip_epsilon=0.01)
- The BTS logging policy creates uneven coverage, partially degrading IPS quality

### Context Sensitivity
- Spearman ρ (perturbation test): **0.70**
- Top-5 overlap after ±0.5σ context perturbation: **62%**
- SAC is the **only context-sensitive agent** in the current benchmark ✅

## 4. Limitations

1. **Offline training only:** SAC was trained from batch data, not via online interaction. Performance in live recommendation may differ.
2. **Context representation:** The current context uses a single scalar `user_item_affinity`. Richer features (demographics, session history) would likely improve personalization.
3. **BTS propensity instability:** Evaluation on BTS scenario has elevated clipping rate (13%), slightly reducing the reliability of IPS/DR estimates.
4. **Item cold-start:** Items not present in training data have no learned Q-values and receive random scores.

## 5. How to Use

### Load from Artifacts

```python
from deeprl_recsys.serving.runtime import ServingRuntime

runtime = ServingRuntime(model_dir="artifacts/models/benchmark_sac_synthetic")
context = {"user_item_affinity": 0.8, "user_id": 42}
candidates = list(range(50))

recommendations = runtime.predict(context, candidates, k=5)
# Returns: [{"item_id": 12, "score": 0.31}, {"item_id": 7, "score": 0.24}, ...]
```

### CLI Evaluation

```bash
deeprl-recsys evaluate --config configs/experiments/exp_benchmark_synthetic.yaml
```
